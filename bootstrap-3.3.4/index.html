Liberty Mutual

LM.com

Node.js

elasticsearch.

Software Dev. Intern ...

Developed a logging workflow that provided real time analysis of LibertyMutual.com's status. Originally, Splunk was used but I advocated the use of the open source framework 'elasticsearch' and collaborated with a second internal team to build it out. The technologies involved included Apache Kafka, elastics Kibana and elasticsearch.

The project was presented to a board of CTOs and CIOs during a tech deep-dive session this past summer. This project utilized a popular open source project and saved Liberty Mutual in proprietary logging costs.

I also developed the 'hub' for developer to access all logs nationwide in a single location through a Node.js application created off of the Express framework as well as ejs for html templating. This abstracted the need to access servers directly to pull log files.

New ESPN app

Software Dev. Intern ...

Developed an optimized data transfer process by moving data processing from mobile devices to ESPN servers. This project was developed in the dynamic language Groovy. Entering the internship I had little knowledge of Groovy or the original protocol that I transferred from (an in-house language derived from C used by ESPN to process JSON).

The result was a program built off the SpringBoot framework and spanned over 7,000 lines of code (over 10,000 with docs/bundled config json/etc) in 10 weeks. The project, named Dark.Binder was inherited by a development team for performance testing and a goal to entirely replace the existing legacy code that runs the backend of the ESPN app and other mobile applications.

Emma5

The Robot

Part of Power Distribution Unit

More Robot

Independent coursework ...

Contributed to an autonomous robot, named Emma5, that was programmed using Arduino devices. The robot is powered by a 12 volt car battery and contains 8 infrared sensors to navigate it's surroundings. We developed the power distribution unit to prevent short circuiting which had occurred in the past.

I was advised by Charles 'Chuck' Malloch, PhD and worked closely with peer Adam Siegel. This project was undertaken my first semester at UMass Amherst. Some cool gadgets I got my hands on include EagleCAD, thermistors (heat resistors), multimeters, oscilliscopes, and too much sodder.

We also hacked an old Nintendo-64 steering wheel and gas pedals to steer the robot manually using a wireless arduino module. I have since left the project but the Electrical Engineering lab at M5 continues to add functionality.

2012 began my defining experience in engineering and computer science through the First Tech Challenge. We designed a 24"x24" robot that could lift objects 10 feet in the air to score points for tournament play. Our team won the Bridgewater State Qualifier and moved onto the State Championship where we placed (a disappointing) second. The robot was programmed in Carnegie Mellon's ROBOTC and was completely fabricated in the high schools machine shop.

After winning at Bridgewater, we had 5 weeks before the State Championship so we made the decision to started from scratch in an attempt to qualify for nationals. We were a single spot out of nationals at the State Championship
